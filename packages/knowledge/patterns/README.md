# Patterns

Cross-cutting **UX and UI patterns** that span multiple components or tokens (flows, layout, content patterns—not a single component’s API).

For component-specific APIs, see [components](../components/). For tokens, see [tokens](../tokens/).

**Patterns** here mean multi-step or multi-surface guidance: forms, navigation, empty states, messaging, accessibility patterns that combine several primitives, and similar topics.

This section should reference those building blocks by name and link to repo or published docs without duplicating full API lists. **Implementation still wins**: if a pattern description conflicts with how components actually behave in code, update the pattern text.
