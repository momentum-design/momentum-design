<h1 class="heading1">Tokens</h1>
<p class="subheading">Design Tokens are variables that represent hard coded values for various types of design decisions; spacing, color, typography, font, elevation, border radius, etc. They can represent anything defined by a designer; a RGBa value for a semi-transparent color, or the pixel width of a border. We use design tokens in the place of hard coded values in order to create flexibility and consistency across all our products within Webex. </p>
<p class="subheading">In Momentum, we have integrated design tokens into our components, assets, and UI toolkits. </p>
<h2 class="heading2">Core</h2>
<p class="subheading">These tokens represent the base values in Momentum. They can be directly used, but are also inherited by all other token types.</p>
